## Definition of "audited" (9 checks)

Each component must pass all nine, verified on **AAPL (US) + BHP.AX (AU) + SHOP.TO (CA)**:

1. **Parity** тАФ matches `reference/original-design.html` (or design-system + owner intent for new/diverging sections).
2. **Data** тАФ displayed values match Supabase raw (`curl` SSR + `execute_sql`).
3. **Calc** тАФ derived values independently recomputed and matched.
4. **Color** тАФ matches `docs/design-system.md` tiers/chart colors.
5. **Layout** тАФ no clipping/overflow at desktop + quick 375px check.
6. **Tooltips** тАФ present and correct.
7. **Empty/insufficient-data** тАФ graceful state on sparse ticker (no fabricated scores).
8. **Beginner clarity** тАФ a non-finance user can understand it.
9. **A11y (light)** тАФ chart `aria-label`, focus-visible.

Status key: тЬЕ pass ┬╖ тЪая╕П issue logged ┬╖ тЭМ fail ┬╖ тмЬ not yet audited ┬╖ ЁЯФз fixed this round
